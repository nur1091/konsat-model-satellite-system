# Design Review Evolution

## Preliminary Design Review

The surviving PDR contains 60 pages. Within the author's assigned scope, it establishes:

- Communication, electrical and ground-station requirements.
- Initial interface breakdowns for the assigned subsystems.
- Processor, RTC, XBee radio and antenna trade studies.
- Telemetry, command, ground-station and power concepts.
- An initial free-space link-budget calculation.

## Critical Design Review

The surviving CDR contains 86 slides. Within the author's assigned scope, it refines the communication, ground-station and electrical-interface baseline.

The CDR file also contains unfinished template slides and does not include measured qualification or end-to-end test results. For that reason, repository claims stop at design and analytical evidence.

## Portfolio treatment

- Personal student numbers, contact details and unused template pages are excluded.
- Full PDR/CDR files are not redistributed.
- Team-wide work is described as team output; Nisanur Kayğusuz's responsibility is limited to the communication system, ground station and electrical systems shown in the team structure.
- RF values are recalculated from the selected components rather than copied from an inconsistent draft calculation.
- Open interfaces and missing verification evidence are stated explicitly.

## Curated visual evidence

The repository retains only standalone source assets that could be extracted directly from the CDR presentation without using full-slide screenshots:

- Payload communication chain.
- Two telemetry-field definition tables.

Low-contrast duplicate diagrams and all visuals outside the author's assigned contribution are excluded from the main README. Component photographs and incomplete electrical slides are also excluded. The retained assets contain no student numbers and are not presented as implementation or qualification results.

## Reconstructed demonstrator

The 1 Hz telemetry demonstrator was created after the design reviews to make the documented interface executable and reviewable. It is labelled throughout the repository as simulated portfolio data and must not be interpreted as original flight software or measured telemetry.
